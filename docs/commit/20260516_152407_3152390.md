# WTPM-CD 性能优化与架构重构

- **Time**: 2026-05-16 15:24:07
- **Commit**: 3152390
- **Branch**: refactor/wtpm-performance

## Features

### 内联数组存储替代堆分配
每个 determinant 的值存储从 `std::vector<double>` 改为 `std::array<double, 2*WTPM_MAX_P>`（MAX_P=32），消除了 hash 插入时的堆分配开销。对于 p=5~20 的典型场景，内联存储使热路径零分配。

### CRTP 模板策略消除虚函数派发
`WTPMWaveFunctionBase` 从虚基类改为非虚基类，`solve()` 通过模板参数 `WFType` 直接调用具体实现（`WTPMWaveFunctionStd` / `WTPMWaveFunctionCuckoo`），编译器可完全内联所有热路径 hash 操作。

### 合并 hash 查找
新增 `get_x_and_xz()` 方法，将坐标选择循环中的 `get_xz()` + `get_x()` 两次查找合并为一次，同时缓存 `x_all` 用于后续 cubic 计算和 `update_x`，消除了冗余的第三次查找。

### 扁平化 S 矩阵存储
S 矩阵从 `vector<vector<double>>` 改为 `std::array<double, WTPM_MAX_P*WTPM_MAX_P>`，以 `S[i*WTPM_MAX_P+j]` 索引，内存连续、缓存友好。`update_x` 中的 S 更新改为基于 `x_k_all` 的单趟计算，去除了 undo/redo 冗余。

### 统一 hash functor 至独立头文件
`DeterminantHash<N>`、`DeterminantHashRobinhood<N>`、`DeterminantEqual<N>` 从 `wavefunction.h` 提取至新文件 `hash.h`，消除了 CDFCI 与 WTPM 代码间的脆弱 include 顺序依赖。

### 移除冗余状态
删除了 `WTPMCD::W` 成员（与 `WTPMWaveFunction::W` 重复），solver 直接从 wavefunction 读取权重。
